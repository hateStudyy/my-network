# 模块 4：代理 / VPN / 翻墙原理

## 共同本质

> 借一个能访问目标的服务器作为中转。

所有翻墙工具都在做这件事。区别在于：
1. 流量走在第几层
2. 是否加密
3. 是否伪装
4. 协议是否动态

## 6 种工具的对比

| 工具 | 工作层 | 加密 | 伪装能力 | 抗 DPI |
|---|---|---|---|---|
| HTTP 代理 | L7 | ❌ | 无 | 极差 |
| SOCKS5 | L5 | ❌ | 无 | 极差 |
| Shadowsocks | L5 | ✅ 对称密钥 | 无规律字节流 | 中 |
| V2Ray (VMess+WS+TLS+CDN) | L5 | ✅ | 伪装成 HTTPS 网站 | 高 |
| WireGuard VPN | L3 | ✅ | 无（定长 UDP 特征明显）| 低 |
| Tor | L5 | ✅ 三层 | 一般 | 中 |

## 为什么 V2Ray + WS + TLS + CDN 是当前最强组合

```
真实流量：[VMess 加密] [WebSocket 帧] [TLS 加密] [TCP/443]

GFW 看到的: 你和 cloudflare.com 建立了一个 https 长连接，
         发了一些加密字节。
```

这种流量和"访问任何 SaaS 应用"的流量完全无法区分。要封它，等于要封掉整个 Cloudflare。

## VPN 和代理的本质区别

- **VPN 在 L3**：操作系统加一块虚拟网卡，所有 IP 包（包括 ping、游戏、DNS）都被加密发到对端
- **代理在 L5+**：只能代理"知道用代理"的应用层协议，TCP/UDP 流量

VPN 是"全家桶"，代理是"按需点单"。

## DPI（深度包检测）做了什么

防火墙不再只看端口/IP，而是分析包的内容特征：
- TLS ClientHello 的 SNI 字段（"我要访问什么域名"）
- 包长度分布、握手时序
- 连接建立后的字节模式

V2Ray 的 WS+TLS 之所以有效，是因为它让前置的 TLS 握手 SNI 看起来是个正常网站，而后续加密内容长度分布也和正常网站一致。

## 当前页面读取了什么

`/api/proxy-status` 调用 `scutil --proxy` 读 macOS 系统代理设置，并读 `http_proxy` 等环境变量。**只读不改**——这是观测工具，不是配置工具。
