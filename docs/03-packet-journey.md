# 模块 2：一个网络包的旅行

## 演示什么

从你在浏览器输入 `https://example.com` 到看到页面，背后发生的 8 步：

1. **DNS 查询**：example.com → IP
2. **TCP 三次握手**：建立连接
3. **TLS 握手**：协商加密
4. **HTTP 请求**：发出 GET
5. **多跳路由**：经过路由器/ISP/骨干
6. **服务器响应**：返回 HTML
7. **页面渲染**：浏览器解析 + 绘制

## 实现思路

- 状态机：8 个步骤的索引
- 视觉：SVG 五个节点（你 → 路由 → ISP → 骨干 → 服务器）+ Framer Motion 让一个圆点根据 step 移动
- 旁边伴随："步骤详情面板"和"OSI 七层高亮"

## 为什么不直接抓真实包来演示？

因为 TLS 握手、DNS 查询、TCP 握手是几毫秒的事，且抓包需要 sudo。"演示"和"教学"用动画足够，**真实数据放在抓包模块**。两个模块互补。

## OSI 七层在这里的体现

每一步标注它发生在哪一层。比如：
- DNS 在应用层
- TCP 三次握手在传输层
- 多跳路由的"换 MAC 不换 IP"在数据链路层和网络层

右侧面板会高亮当前层。
